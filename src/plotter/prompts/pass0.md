# Промпт: Pass 0 — Определение контекста сериала

> **Самостоятельный документ.** Подаётся в LLM как есть. Самый дешёвый промпт в пайплайне — мало токенов на входе, быстрый ответ.

## Контракт

- **Вход**: название шоу, номер сезона, 2–3 первых синопсиса (+ описание шоу, если есть)
- **Выход**: JSON с franchise_type, story_engine, genre, format → передаётся как вход в Pass 1 и Pass 2

## Вход

- **show**: название шоу
- **season**: номер сезона
- **description**: описание шоу / logline / premise (если есть, иначе пустая строка)
- **sample_synopses**: 2–3 первых синопсиса сезона (текст)

## Задача

Определи два параметра: franchise type и story engine. Они определяют как пайплайн будет извлекать сюжетные линии.

### Шаг 1: Franchise type

Определи структуру эпизодов по паттерну первых синопсисов:

| type | признак | пример |
|------|---------|--------|
| `procedural` | Каждый эпизод содержит самостоятельную историю (дело, пациент, задание), которая открывается и закрывается внутри эпизода | House, CSI, Law & Order |
| `serial` | Эпизоды продолжают друг друга, конфликты не закрываются внутри эпизода | Breaking Bad, The Americans, The Wire |
| `hybrid` | Есть самостоятельная история эпизода (case-of-week) + сквозные линии между эпизодами | X-Files, Buffy, Castle |
| `ensemble` | Несколько равноправных персонажей, у каждого своя линия, нет единого протагониста | Game of Thrones, This Is Us, The Crown |

Если E01 и E02 имеют разные «дела недели» — procedural или hybrid. Если E02 продолжает конфликт E01 без закрытия — serial. Если у каждого эпизода свой фокусный персонаж — ensemble.

### Шаг 2: Story engine

Выбери ближайший шаблон для определённого franchise type и заполни слоты:

**Procedural:**
> Каждую неделю [профессия] решает [тип задачи], проверяя [качество героя]

Примеры: «Каждую неделю диагност решает медицинскую загадку, проверяя границы этики» (House). «Каждую неделю следователи раскрывают убийство, проверяя как далеко зайдёт система правосудия» (Law & Order).

**Serial:**
> [Герой] [трансформация], проверяя как далеко он готов зайти ради [цель]

Примеры: «Школьный учитель строит наркоимперию, проверяя как далеко он готов зайти ради семьи и контроля» (Breaking Bad). «Пара шпионов ведёт двойную жизнь, проверяя как далеко они готовы зайти ради идеологии» (The Americans).

**Hybrid:**
> Каждую неделю [задача], на фоне [сквозной конфликт]

Примеры: «Каждую неделю новое паранормальное дело, на фоне правительственного заговора» (X-Files). «Каждую неделю демоны и вампиры, на фоне взросления и отношений» (Buffy).

**Ensemble:**
> [Группа] [общая ситуация], каждый проживая [свой аспект темы]

Примеры: «Семья через три поколения, каждый проживая свой аспект идентичности и потери» (This Is Us). «Знатные дома борются за трон, каждый проживая свой аспект власти и чести» (Game of Thrones).

## Формат выхода

Ответ — строго JSON, без markdown-обёртки, без комментариев вне JSON.

```json
{
  "show": "Breaking Bad",
  "season": 1,
  "franchise_type": "serial",
  "story_engine": "Школьный учитель строит наркоимперию, проверяя как далеко он готов зайти ради семьи и контроля",
  "genre": "drama",
  "format": "ongoing",
  "reasoning": "Эпизоды продолжают друг друга: конфликт E01 (первая варка) перетекает в E02 (последствия), нет закрытых историй внутри эпизодов"
}
```

### Типы полей

- `show`: string
- `season`: integer
- `franchise_type`: enum — `"procedural"` | `"serial"` | `"hybrid"` | `"ensemble"`
- `story_engine`: string — заполненный шаблон (одно предложение)
- `genre`: string — `"drama"`, `"thriller"`, `"comedy"`, `"sci-fi"` и др.
- `format`: enum — `"ongoing"` | `"limited"` | `"anthology"` | null (если неясно)
- `reasoning`: string — краткое обоснование выбора franchise_type (1–2 предложения)

## Валидация

Валидация выполняется кодом, не LLM:
- JSON-схема: все обязательные поля, enum-значения
- `franchise_type` — одно из четырёх значений
- `story_engine` — не пустая строка

После валидации результат показывается человеку для подтверждения или правки.
