# ADR-005: Pass 3 — ревизия нарратолога

## Статус
Проектирование

## Контекст
Pass 2 генерирует патчи (ADD_LINE, CHECK_LINE, SPLIT_LINE, RERANK) — обратную связь о проблемах в списке линий из Pass 1. Сейчас патчи собираются, но не применяются.

Первоначальный план — перезапустить Pass 1 с учётом патчей. Проблемы:
- Дорого: повторная обработка всех синопсисов
- Рискованно: LLM может сломать то, что уже правильно
- Реактивно: работает только с тем, что Pass 2 заметил

## Решение
Отдельный Pass 3 с ролью **нарратолога** — «второе мнение» от другого взгляда.

### Три пасса — три взгляда
- **Pass 0** — генералист (контекст сериала)
- **Pass 1 + Pass 2** — аналитик (линии + события)
- **Pass 3** — критик/нарратолог (ревизия результата)

### Что видит нарратолог
- Линии + каст (Pass 1)
- События по эпизодам (Pass 2)
- Computed span и weight (пост-обработка)
- Патчи из Pass 2 как *подсказки*, не как задания
- Story engine и franchise type для контекста

### Какие вопросы задаёт
Не «проверь на ошибки», а теоретические вопросы:
- У каждой линии есть driver → goal → obstacle → stakes? Если нет — это не storyline, а thread
- Арка полная? setup без escalation — мертворождённая линия
- Два storyline с одним driver и смежными целями — одна линия с фазами?
- Computed weight = "primary", rank = "C" — кто прав?
- Franchise type = hybrid, но нет episodic-линии — что-то не так
- ADD_LINE: реально новая линия, или ошибка привязки события?

### Выход — вердикты (не патчи)
Вердикты достаточно конкретны, чтобы код применил их механически:
- `MERGE(line_x → line_y)` — все события line_x → line_y
- `REASSIGN(event, → line_z)` — точечное переназначение
- `PROMOTE/DEMOTE(line, new_rank)` — смена ранга
- `CREATE(new_line {driver, goal, ...}, events: [...])` — новая линия + какие события ей принадлежат
- `DROP(line, redistribute: {event → line, ...})` — удаление линии с перераспределением событий

### Применение
Код применяет вердикты к линиям и событиям. Без повторного Pass 1 или Pass 2.

RERANK можно вычислить кодом (из весов) до Pass 3 — нарратолог его не касается.

### Ограничения
- Один вызов LLM, один раунд ревизии
- Если после Pass 3 остаются нерешённые вопросы — оставляем как есть с `confidence: "inferred"`

## Что нужно проверить
1. Достаточно ли контекста нарратологу без сырых синопсисов? (видит только линии + события + span/weight)
2. Хватит ли одного вызова, или объём данных (все эпизоды, все линии, все события) не влезет в контекст?
3. Формат вердиктов: сможет ли LLM стабильно генерировать структурированные операции (MERGE, REASSIGN, CREATE)?
4. Применение вердиктов кодом: покрывают ли 5 операций все реальные кейсы?
5. Качество: действительно ли «второе мнение» улучшает результат vs просто добавляет шум?
