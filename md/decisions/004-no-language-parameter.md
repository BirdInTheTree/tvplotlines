# ADR-004: Без параметра языка — вход определяет выход

## Статус
Принято

## Контекст
Первоначально API предполагал параметр `language` (или `output_language`). Но зачем? Если пользователь подаёт синопсисы на русском — он хочет результат на русском. Если на немецком — на немецком.

## Решение
Никакого параметра языка. LLM определяет язык входа и отвечает на нём же. В промпте: "Respond in the same language as the input."

```python
# Просто
result = get_plotlines(episodes=["синопсис 1", "синопсис 2", ...])
```

## Варианты
1. **`language="en"` обязательный** — ограничивает аудиторию, добавляет сложность
2. **`output_language` отдельно от входа** — зачем? кому нужны синопсисы на русском с ответом на английском?
3. **Без параметра, язык = язык входа** ← выбрано

## Почему именно так
- Минимальный API — меньше параметров, проще пользоваться
- Не ограничивает аудиторию одним-двумя языками
- LLM справляется с определением языка без подсказок
- Синопсисы можно искать на любом доступном языке — библиотека работает со всеми

## Последствия
- Промпты пишутся language-agnostic: "Respond in the same language as the input"
- Не нужны отдельные файлы промптов для EN/RU (как в mas4bw)
- Если пользователю нужен другой язык выхода — это edge case, пусть переводит сам
