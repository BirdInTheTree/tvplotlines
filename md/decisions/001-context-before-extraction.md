# ADR-001: Определять контекст сериала до экстракции линий

## Статус
Принято

## Контекст
В mas4bw (оригинал Balestri & Pescatore) экстракция арок начинается сразу, без понимания что за сериал перед нами. Результат: в "Слово пацана" S01 извлечено 25 арок (10 Anthology, 7 Soap, 8 Genre-Specific), хотя в реальности основных линий 3.

## Решение
Ввести Шаг 0 — автоматическое определение контекста сериала перед экстракцией:

**0a. Franchise type** (procedural / serialized / hybrid)
- Определяет СТРУКТУРУ ожидаемых линий
- Procedural → A-story закрытая (case-of-week), B/C сериализованные
- Serialized → все линии сквозные
- Hybrid → A закрытая + B/C сквозные

**0b. Story engine** (что генерирует эпизоды)
- Определяет КАКИЕ линии искать
- Case-of-week engine → линия расследования/пациента/дела
- Transformation machine → линия трансформации героя
- Relationship engine → линии отношений

Также определяется жанр и формат.

## Варианты
1. **Пользователь задаёт контекст вручную** — точнее, но требует экспертизы
2. **ИИ определяет автоматически** — удобнее, один дополнительный LLM-вызов
3. **Оба варианта** ← выбрано. Автоопределение по умолчанию, ручной ввод как override

## Почему именно так
- Без контекста экстрактор не знает сколько линий ожидать и какого типа
- Один LLM-вызов на все синопсисы сразу — дёшево
- Ручной override нужен когда пользователь знает лучше (или для тестирования)

## Последствия
- `get_plotlines()` возвращает `PlotterResult` с полем `context` — пользователь видит что определилось
- Шаги экстракции получают контекст как параметр и адаптируют промпты
- Промпты для экстракции должны учитывать franchise type (разные ожидания для разных типов)
