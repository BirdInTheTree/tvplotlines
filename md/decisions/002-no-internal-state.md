# ADR-002: Никакого скрытого состояния внутри библиотеки

## Статус
Принято

## Контекст
В mas4bw шаг 2 графа экстракции открывает сессию SQLite и читает ранее найденные арки из базы данных. Также `initialize_state` читает 4 файла с диска. Это делает код непереносимым: нельзя вызвать как функцию, нужен работающий SQLite + файловая система.

## Решение
Библиотека — чистая функция. Всё на вход, всё на выход. Внутри нет:
- Базы данных (SQLite, ChromaDB)
- Чтения/записи файлов
- Скрытого состояния между вызовами
- Веб-сервера

Пользователь подаёт ВСЕ синопсисы сезона в один вызов `get_plotlines()`. Функция сама проходит эпизоды последовательно и накапливает линии in-memory. Состояние живёт только во время вызова и не сохраняется между вызовами.

## Варианты
1. **Встроенная БД** (как в mas4bw) — удобно для приложения, невозможно для библиотеки
2. **Файловый кэш** — неявная зависимость от файловой системы
3. **`known_plotlines` параметр** — перекладывает сложность на пользователя, он не хочет этим заниматься
4. **Весь сезон в один вызов, состояние in-memory** ← выбрано

## Почему именно так
- Библиотеку можно вызвать из Jupyter notebook, из CLI, из веб-сервера, из другой библиотеки
- Пользователь контролирует хранение (может вообще не хранить)
- Тестируемость: вход → выход, без side effects
- Урок из техаудита mas4bw: два места привязки к инфраструктуре — главный блокер для переиспользования

## Последствия
- Пользователь подаёт весь сезон одним вызовом — проще некуда
- Внутри функция сама итерирует по эпизодам и накапливает контекст
- Нет персистентного состояния — каждый вызов независим
- Инкрементальная обработка (серия за серией) — возможный low-level API в будущем, не приоритет
