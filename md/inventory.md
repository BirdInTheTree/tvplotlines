# Инвентаризация: теория, технология, своё, чужое

## 1. ТЕОРИЯ (знания, модели, классификации)

### Своё
- **A/B/C классификация сюжетных линий** — вместо академических anthology/soap/genre → индустриальные A (жанр), B (отношения), C (побочная). Основано на Nash, Douglas, реальной практике
- **Критика оригинала** — "25 арок в Слово пацана — это бред, обычно 3 основных линии". Понимание что модель перегенерирует
- **Procedural / serialized / hybrid** — типология форматов сериалов, влияет на то какие линии ожидать
- **Horizontal / vertical storylines** — из Douglas, деление линий по способу развёртывания
- **Формула "знания из книг → формальная модель → инструкции для ИИ"** — методологический подход, которого нет у авторов

### Из книг (база знаний, строится сейчас)
- Nash — Save the Cat Writes for TV (A/B/C storylines, 15-beat)
- Douglas — Writing the TV Drama Series (horizontal/vertical, структура сезона)
- Landau — TV Showrunner's Roadmap (21 навигационная точка)
- Mittell — Complex TV (нарративная сложность)
- Weiland — Creating Character Arcs
- Rabkin — Writing the Pilot
- + ещё ~14 книг в how2pitch по питчу и структуре

### Из статей
- Balestri & Pescatore 2025 (ICAART) — идея мультиагентной экстракции арок
- Статьи с arxiv — решения проблем контекста и галлюцинаций (конкретные?)

## 3. ЧТО НУЖНО ПЕРЕПИСАТЬ ДЛЯ СВОЕЙ РЕАЛИЗАЦИИ

Всё что под лицензией CC BY-NC-ND (Balestri & Pescatore) нужно написать заново.
Идеи из статьи использовать можно — идеи не защищаются лицензией.
Код — нельзя копировать.

### Что берём как ИДЕЮ (из статьи):
- Мультиагентный подход к экстракции (несколько LLM-вызовов с разными ролями)
- Дедупликация через векторное сходство
- Верификация результатов отдельным шагом

### Что пишем СВОЁ (на основе книг + опыта):
- Классификация линий: A/B/C вместо anthology/soap/genre
- Количество и логика шагов: не 9 фиксированных, а столько сколько нужно
- Промпты: на основе знаний из 20 книг, а не эмпирического подбора
- Схема данных: Storyline вместо NarrativeArc, с полями под A/B/C модель
- Входной формат: синопсисы как основной путь (не полные тексты)

## 4. OPEN SOURCE АРХИТЕКТУРА

Из трёх продуктов mas4bw — лучший кандидат на библиотеку/фреймворк.
books2series — был назван лучшим кандидатом на фреймворк (расширяемые схемы, предикаты, entity resolution).

### Вариант: единый фреймворк

```
screenwriting-toolkit/              ← зонтичный проект
├── core/                           ← общие модели, утилиты
│   ├── models.py                   ← Storyline, Character, Event, Episode
│   ├── llm.py                      ← абстракция над LLM-провайдерами
│   └── io.py                       ← чтение синопсисов, книг, PDF
├── storylines/                     ← бывший mas4bw
│   ├── extract.py                  ← extract_storylines(episodes) → list[Storyline]
│   ├── classify.py                 ← A/B/C классификация
│   ├── prompts/                    ← промпты по языкам
│   └── knowledge/                  ← знания из книг (что такое A-line, B-line)
├── books/                          ← бывший books2series
│   ├── extract.py                  ← extract_events(book) → list[Event]
│   ├── characters.py               ← extract_characters(book) → list[Character]
│   ├── screen.py                   ← screen_book(book, brief) → BookCard
│   └── domain/                     ← предикаты, онтологии
├── pitch/                          ← бывший how2pitch
│   ├── knowledge/                  ← база знаний из 20 книг
│   ├── wizard.py                   ← пошаговый визард
│   └── templates/                  ← шаблоны питч-библии
└── apps/                           ← веб-интерфейсы (не open source?)
    ├── storyline_viewer/
    ├── book_explorer/
    └── pitch_wizard/
```

### Что открываем, что нет

| Компонент | Open source? | Почему |
|---|---|---|
| core/ | Да | Общие модели, всем полезны |
| storylines/ | Да | Переписано с нуля, на основе книг |
| books/ | Частично | Базовая экстракция — да, domain-модели — нет |
| pitch/knowledge | Да (датасет) | Люди добавляют книги |
| pitch/wizard | Нет | Продукт, можно продавать |
| apps/ | Нет | Коммерческие интерфейсы |
