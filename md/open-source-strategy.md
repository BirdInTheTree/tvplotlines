# Open Source стратегия

## Главный принцип: библиотека vs приложение

```
Библиотека (open source)          Приложение (не open source)
─────────────────────────         ─────────────────────────
get_plotlines() → результат       БД + интерфейс + редактирование
чистое вычисление                 хранение, показ, работа с данными
pip install plotter               твой продукт
```

Аналогия: калькулятор бесплатный, Excel платный.

## Что открываем, что нет

| Слой | Open source? | Почему |
|---|---|---|
| Библиотека `get_plotlines` | Да, MIT | Вычислительное ядро, привлекает пользователей и контрибьюторов |
| База знаний (how2pitch) | Да, как датасет | Люди могут добавлять книги, переводы, карточки |
| Приложение (БД + UI + редактирование) | Нет | Коммерческая ценность, продукт автора |
| books2series | Нет (пока) | Главное конкурентное преимущество |

## Кому нужна библиотека

- **Исследователи** (computational narratology) — вызвать из Jupyter на своём корпусе
- **Разработчики** — встроить в свои инструменты для сценаристов
- **Студии** — интегрировать во внутренний пайплайн

## Что нужно для выпуска

1. Написать ядро с нуля (не форк mas4bw, новая реализация на основе книг)
2. Оформить как Python-пакет с чистым API
3. README с примерами использования
4. Опубликовать на PyPI + GitHub
5. Лицензия: MIT

## Открытые вопросы

- Готовность поддерживать пакет (issues, PR, документация) — отдельная работа
- Если не готова — выложить на GitHub с MIT без обещания поддержки
- Какой Reddit: r/screenwriting (~1M), r/LanguageTechnology, r/MachineLearning
